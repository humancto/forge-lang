# Forge RFCs

This directory contains Request for Comments (RFC) documents that describe the design decisions behind Forge.

RFCs explain **why** things work the way they do â€” not just what they do. They serve as a historical record of design intent and as a guide for future contributors.

## How to Submit an RFC

1. Copy the template below into a new file: `rfcs/NNNN-short-title.md`
2. Fill in the sections
3. Open a pull request

## RFC Template

```markdown
# RFC NNNN: Title

- **Status:** draft | accepted | implemented | rejected
- **Author:** Your Name
- **Date:** YYYY-MM-DD

## Summary

One paragraph explaining the proposal.

## Motivation

Why is this change needed? What problem does it solve?

## Design

How does it work? Include syntax examples, API signatures, and behavior descriptions.

## Alternatives Considered

What other approaches were evaluated? Why were they rejected?

## Implementation Notes

How should this be implemented? What files need to change?
```
